# md2pptx

**Markdown と PowerPoint テーマ（thmx / pptx）から発表スライド（pptx）を生成するツール。**

スライドのデザインのおおもとは PowerPoint テーマであり、配色・フォントはテーマに委ねます
（スクリプト側で色やフォントをハードコードしません）。内容は書きやすい Markdown で記述し、
行頭の見た目から段落種別（箇条書き・採番・結論行）を自動判定します。表・フロー図・2カラム・
タイトル改行にも対応します。

```
theme.thmx  ┐
            ├─▶  md2pptx  ─▶  out.pptx
input.md    ┘
```

## 特徴

- **テーマ駆動**：thmx（または pptx）の配色・フォント・レイアウトをそのまま使用
- **行頭マーカー記法**：`-` 箇条書き / `1.`・`①`・`(1)` 採番 / `→` 結論行 を見た目で自動変換
- **表**：Markdown 標準テーブル（ヘッダ着色・列幅指定）
- **フロー図**：独自の ` ```flow ` ブロックで box・矢印・キャプションを宣言的に記述
- **2カラム**：`<!-- @col -->` で「2つのコンテンツ」レイアウト
- **発表者ノート**：` ```note ` ブロックでオーラル原稿をスライドと一緒に版管理
- **相対サイズ**：`{+1}`/`{-2}` で行ごとに強調・縮小（テーマ基準の相対段数のみ。絶対 pt は持たない）
- **自動調整**：本文の標準フォントサイズに揃え、収まらない表・図は下位レベルへ段階縮小
- **はみ出し防止**：地の文は標準プレースホルダに配置し、スライド外へあふれない

## 必要環境

- Python 3.9+
- [python-pptx](https://python-pptx.readthedocs.io/) 1.0.2（インストール時に自動導入）
- [PyYAML](https://pyyaml.org/)（同上）

## インストール

`md2pptx` コマンドとして使えるようにするには、リポジトリのルートで次のいずれかを実行します。

```bash
# 推奨：pipx で隔離環境へインストール（依存も自動導入・PATH に md2pptx が入る）
pipx install .

# もしくは通常の pip（venv 推奨）
pip install .

# 開発用（編集が即反映される editable インストール）
pip install -e .
```

> pipx が無い場合は `brew install pipx`（macOS）や `python3 -m pip install --user pipx`
> で導入できます。インストールせずに試すだけなら、依存（`pip install python-pptx pyyaml`）を
> 入れたうえで `python3 -m md2pptx …` でも実行できます。

## 使い方

```bash
# テーマ（.thmx か .pptx）と Markdown を指定して pptx を生成
md2pptx input.md --theme theme.thmx -o out.pptx

# テーマ・出力先はフロントマターにも書ける（CLI 引数が優先）
md2pptx input.md

# インストールせず開発中に実行する場合
python3 -m md2pptx input.md
```

| オプション | 説明 |
|---|---|
| 位置引数 | 入力 Markdown ファイル |
| `--theme PATH` | テーマファイル（`.thmx` / `.pptx` 両対応）。フロントマター `theme:` を上書き |
| `-o, --output PATH` | 出力 pptx。フロントマター `output:` を上書き |
| `--keep-base PATH` | `.thmx` から作る中間 base pptx を破棄せず保存（デバッグ用） |
| `--version` | バージョンを表示して終了（入力ファイルは不要） |

`.thmx` を渡すと内部で base pptx に変換してから描画します（テーマ更新が即反映）。
`.pptx` を渡すとそのまま土台に使います（変換をスキップ）。

### 試す

```bash
md2pptx example.md --theme OfficeTheme.pptx -o example.pptx
```

## Markdown 文法

### フロントマター（YAML）

ファイル冒頭を `---` で囲み、テーマ・出力先・タイトルスライド情報を書きます。
**`---`…`---` の内側は YAML として、それ以降は Markdown 本文として解釈されます**
（両者は別のパーサが処理するため、記法上の制約も異なります。後述の `{` の注意を参照）。

```yaml
---
theme: OfficeTheme.pptx   # デザインのおおもと（.thmx / .pptx）
output: out.pptx
slide_number: true                 # タイトル以外に番号を付与（既定 true）
default_autofit: true              # 本文があふれる場合に縮小（既定 true）

# タイトルスライド（あれば 1 枚目に生成）
title: |
  md2pptx
  Markdown でつくるスライド
subtitle: ― テーマ駆動のスライド生成 ―
author: md2pptx demo
affiliation:
  - Markdown ＋ PowerPoint テーマ → pptx
  - Python / python-pptx / PyYAML
---
```

- 使えるキーは上記の 8 つ（`theme` / `output` / `slide_number` / `default_autofit` /
  `title` / `subtitle` / `author` / `affiliation`）のみです。**未知のキーはタイポとみなして
  エラーで停止**します。
- `title` の改行は多段タイトルになります。`subtitle` はタイトル枠内に少し小さめで入ります。
- `subtitle` / `author` / `affiliation` の各行は、先頭に本文と同じ相対サイズトークン
  `{+1}` / `{-1}`（後述）を置けます（例: 日付だけ一段小さく）。ただし **YAML では値が `{` で
  始まるとエラーになる**ため、トークン付きの行は `"…"` で囲んでください。
  例: `- "{-1} 2026年7月"`（本文中の `{+1}` はクォート不要です）。

### スライド分割とタイトル

- `## 見出し` で新しいスライドを開始（見出し＝タイトル）
- `---`（水平線）でタイトルなしスライドを開始
- `# 見出し`（H1）はセクション見出しスライド
- `###` 以降（H3〜H6）は**未定義でエラー**（将来のスライド内小見出し用に予約）
- タイトル内に `<br>` を書くとその位置で改行
- 本文行（箇条書き・連番・矢印行など）内の `<br>` も同様にその位置で改行
  （段落内改行。続きの行は箇条書きのぶら下げ位置に揃います）

```markdown
## 行頭マーカー記法：<br>見た目で段落の種別を判定
```

### 本文：行頭マーカー記法

インデント（半角スペース2つ＝1レベル）でネストを表します。

| 書き方 | 表示 |
|---|---|
| `- テキスト` / `* テキスト` | 箇条書き（テーマ既定の記号） |
| `1. テキスト` | 自動採番 `1. 2. 3.` |
| `① テキスト` | 丸数字採番 `① ② ③`（番号文字は除去し自動採番に変換） |
| `(1) テキスト` | 丸括弧採番 `(1) (2)` |
| `→ テキスト` | 行頭記号なしの結論・補足行（`→` は表示に残る） |

- 丸数字マーカーは `①`（U+2460）〜`⑳`（U+2473）を認識します。他のマーカーと違い
  直後のスペースは省略できます（`①本文` も可）。
- `→` 行は「`→` ＋スペース1つ＋本文」へ正規化されます。

採番行の直下にネストした `-` を置くと「見出し＋説明」になります。

```markdown
1. parser.py
   - Markdown を中間表現（IR）へ変換
2. render.py
   - IR を pptx に描画
```

#### 行ごとの相対フォントサイズ

特定の行だけ強調・縮小したいときは、マーカー直後・本文直前に `{+1}` / `{-2}` を置きます。
**絶対 pt は指定できません**。テーマ既定サイズからの相対段数（1 段 ≒ ±12.5%）だけを指定でき、
テーマを差し替えても比率が保たれます。

```markdown
- {+1} ここだけ一段大きく強調
- 通常の行
- {-2} 注記は二段小さく
① {+1} 採番行も大きくできる
→ {-1} 結論行も小さくできる
```

- 全行種（`-` / `1.` / `①` / `(1)` / `→`）で使えます。符号は省略可（`{2}` ＝ `{+2}`）。
- スライド全体を一律で調整するには `<!-- @body-size: -1 -->`（後述）。行の `{…}` が優先されます。
  `{0}` はスライドの `@body-size` を打ち消して「その行だけテーマ既定へ戻す」意味です
  （`@body-size: 0` は未指定と同義）。
- 見栄えはテーマに委ねる方針のため、これは“逃げ道”。常用せず強調したい一点に絞るのが推奨です。

### 表

Markdown 標準のテーブル記法。1 行目をヘッダとしてアクセント色で着色します。

```markdown
## 課題の整理とアプローチ
<!-- @table-widths: 45,55 -->

体裁はテーマに、内容は Markdown に分離する

| 課題 | md2pptx のアプローチ |
|:--|--:|
| デザインの一貫性 | PowerPoint テーマに委譲 |
| 記述のしやすさ | Markdown の行頭マーカー記法 |

→ 色やフォントはコードに持たず、テーマに委ねる
```

- 導入文・結論文（地の文）は本文プレースホルダに入り、表はその中央帯に配置されます。
- `<!-- @table-widths: 45,55 -->` で列幅の比率を指定できます（既定は均等）。
- 区切り行のコロンで列ごとの水平寄せを指定できます：`:--:` 中央 / `--:` 右 /
  `:--` 左（コロン無し `---` も左）。数値列の右寄せに便利です。
- 帯に収まらない表はフォントを本文スタイルの下位レベルへ段階縮小して収めます。
  最小レベルでも収まらない場合は警告を出します（そのまま帯を超過しえます）。
  縮小せず標準サイズのまま下へはみ出したいときは `<!-- @overflow: true -->`
  （後述の「はみ出しの許可」）。

### フロー図

` ```flow ` フェンスブロックで box・矢印のフロー図を宣言的に記述します。

````markdown
## 生成パイプライン

```flow
direction: lr
note(top): テーマと Markdown を入力に pptx を生成
[theme.thmx | テーマ]
-変換-> [base.pptx | 土台]
-描画-> [out.pptx | スライド]
-> [… | ]
caption: 配色・フォントはテーマ、内容は Markdown
note(bottom): → テーマを差し替えるだけで見た目が一新できる
```
````

| 要素 | 記法 |
|---|---|
| ノード | `[ラベル \| サブラベル]`（サブラベル省略可） |
| 省略記号 | `[…]` または `[...]`（box ではなく中央寄せの注記） |
| 矢印 | `->` / ラベル付き `-PR->` |
| 方向 | `direction: lr`（左→右、既定）/ `tb`（上→下）。それ以外の値はエラー |
| 図のキャプション | `caption: …`（box の直下） |
| 上下の地の文 | `note(top): …` / `note(bottom): …`（本文プレースホルダへ） |
| 色の個別指定 | `[名前 \| サブ]{accent6}`（色名は後述のテーマ色名） |

ノード・矢印・設定行のいずれにも解釈できない記述はタイポとみなしてエラーで停止します。
色名に使えるのはテーマ色名 `accent1`〜`accent6` / `tx1` / `tx2` / `bg1` / `bg2` です
（`@autonum-color` と共通。無指定時はテーマから自動割り当て）。

### 画像（jpg / png）

画像も表・フロー図と同じ「オブジェクト」として扱われ、導入文・結論文（地の文）と一緒に
1枚のスライドに置けます。記法は2形式あります。

**ショートハンド（標準 Markdown 画像＋末尾 `{...}`）**

```markdown
![実験結果の比較](results.png){width=70%}
![](diagram.jpg){width=8cm align=left}
```

**フェンス ` ```image `（オプションが多いとき）**

````markdown
```image
src: results.png
width: 70%
crop: 100,50,800,400
align: center
caption: 実験結果の比較
```
````

| キー | 意味 |
|---|---|
| `src` | 画像ファイル（Markdown からの相対パス）。必須 |
| `width` / `height` | 埋め込みサイズ。`70%`（帯幅比）/ `8cm` / `300pt` / `2in`（`2"` も可）/ `150px` / 単位なし＝px。省略時はアスペクト自動 |
| `crop` | トリミング。**残す矩形 `x,y,w,h`**（左上原点）。既定はピクセル、各値に `%` を付けると割合 |
| `align` | 水平寄せ `left` / `center`（既定）/ `right` |
| `fit` | `width` と `height` を両方指定したとき `contain`（既定・比維持）/ `fill`（歪ませ充填） |
| `caption` | 図下キャプション（ショートハンドは alt テキストを採用） |
| `overflow` | `true` で帯からのはみ出しを許可（`width`/`height` の明示指定が必須・無指定はエラー。はみ出すのは下方向のみ）。未指定時はスライドの `@overflow` に従い、指定時はそちらより優先 |

- 画像パスは Markdown ファイルの置き場を基準に解決します。見つからない場合はエラーで停止します。
- 上記以外のキーはタイポとみなしてエラーで停止します。
- ショートハンドの `{...}` は空白区切りのため、空白を含むキャプションは alt テキスト
  （`![実験 結果](...)`）かフェンス記法の `caption:` を使ってください。
- `crop=100,50,800,400` は「左上 (100,50) から 幅800×高400 px の範囲を残す」の意味です
  （`crop=10%,5%,50%,40%` のように `%` でも指定できます）。
- 画像は自動的にアスペクト比を保ち、帯（中央領域）からはみ出さないよう収まります。
  `overflow: true` を指定すると明示した `width`/`height` を優先し、帯の下（結論文・罫線側）への
  はみ出しを許可します（例: `height: 115%` で帯高の 115%。上方向のタイトル・導入文には重なりません。
  `width`/`height` のどちらも無い場合はエラー、スライド下端を超える場合は警告を出力します）。
  指定サイズが帯に収まる場合（例: `height: 80%`）は通常配置と同じで、はみ出しは発生しません。

### 発表者ノート（` ```note `）

` ```note ` ブロックの中身が、そのスライドの発表者ノート（PowerPoint の
発表者ビューやノート印刷で見えるテキスト）になります。スライド面には表示されません。
オーラル原稿をスライドソースと同じファイルで版管理する用途を想定しています。

````markdown
## 背景：LaTeX論文執筆が抱える課題

- 学生側の障壁
  - OS差・TeX環境・パッケージ依存で環境構築が困難

```note
まず背景です．LaTeX の執筆環境は OS 差で環境構築が難しく，
指導のフィードバックも属人化しがちでした．
```
````

- スライド内ならどこに書いても構いません（読みやすさの点では末尾を推奨）。
  複数書くと改行で連結されます。ブロック内の改行は段落区切りとして保持されます。
- 最初の見出しより前（本文開始前）に置くと**タイトルスライドのノート**になります
  （フロントマターに `title:` が必要です）。
- ` ```note ` を解釈しない旧版の md2pptx でもエラーにはならず、単に無視されます。

### 2 カラム（「2つのコンテンツ」）

`<!-- @col -->` を1つ置くと、その前後が左右2カラムに分かれ、自動的に
「2つのコンテンツ」レイアウトになります。

```markdown
## 比較：手作業 と md2pptx

- 手作業でスライド作成
  - 体裁の調整に時間がかかる
  - デザインがばらつきやすい

<!-- @col -->

- md2pptx で生成
  - 内容（Markdown）に集中できる
  - テーマで見た目を統一
```

各カラムには箇条書き・採番だけでなく、**表・フロー図・画像**も置けます。地の文（箇条書き）と
表・図を同じカラムに混在させても重ならないよう配置され、カラムに表・図しか無ければカラム全体に
配置されます。左に要点、右に縦長の推移表、といったレイアウトに使えます。

### ディレクティブ（任意の上書き）

行頭マーカーで表せない指定は、スライド先頭の HTML コメントで補います。

| ディレクティブ | 効果 |
|---|---|
| `<!-- @autonum-color: tx1 -->` | このスライドの採番記号色を指定（テーマ色名 `accent1`〜`accent6` / `tx1` / `tx2` / `bg1` / `bg2`） |
| `<!-- @layout: 5 -->` | レイアウト番号を上書き |
| `<!-- @autofit: 90 -->` | 本文を 90% に縮小して焼き込む |
| `<!-- @body-size: -1 -->` | このスライドの本文を一律で相対調整（行の `{+1}` が優先） |
| `<!-- @table-widths: 45,55 -->` | 表の列幅比率（多カラムではスライド内の全カラムの表に共通適用） |
| `<!-- @col -->` | カラム区切り（2つのコンテンツ）。値は取らない |
| `<!-- @widths: 55,45 -->` | プレースホルダ幅（標準の使用可能幅に対する%）。値1個＝単カラム本文幅（例: `104`）、複数＝カラムごとの幅。合計>100 で全体が拡幅 |
| `<!-- @overflow: true -->` | このスライドの表・画像に、帯（中央領域）から下方向へのはみ出しを許可（後述） |

- **未知のディレクティブはタイポとみなしてエラーで停止**します（行番号付き）。
- `@widths` の拡幅は**左端固定・右方向のみ**が既定（行頭位置がスライド間で揃う）。
  右余白で収まらない指定はクランプ＋警告になります。値の末尾に `!` を付ける
  （例: `108!` / `62,47!`）と、収まらない分だけ左余白へ逃がすことを許可します
  （スライド端は余白 0.1in でクランプ）。
- 値の区切り・記号は全角（`，` `％` `！`）でも受理します。キー名のハイフンは
  アンダースコアと同一視します（`@body-size` ＝ `@body_size`）。
- 旧名称 `@ph-widths` / `@body-width`（→ `@widths`）と `@col-widths`（→ `@table-widths`）
  は v0.7 で廃止されました（エラーメッセージで新名称を案内します）。

#### はみ出しの許可（`@overflow`）

`<!-- @overflow: true -->` を置くと、そのスライドの**表・画像**は「帯（中央領域）に
収める」制約を外せます。共通の規約:

- **上端は帯の上端に固定**され、はみ出すのは**下（結論文・罫線側）のみ**。
  タイトル・導入文には重なりません。
- **表**: フォントを段階縮小せず、**本文標準（lvl1）サイズのまま**必要な高さで
  描画します。行数の多い表が読めないほど小さくなるのを防げます。
- **画像**: スライド既定として作用します。画像ブロックの `overflow:`（per-block）
  を明示した場合は**そちらが優先**されます。サイズ（`width`/`height`）の明示が
  ない画像は帯に内接するため、`@overflow` を置いても変化しません。
- 帯に収まる表・画像には効果がありません（通常配置と同じ）。
- **結論文との重なりは仕様**です（結論文の帯に意図的に重ねる機能）。実際に
  重なる場合とスライド下端を超える場合は stderr に警告を出します。
- フロー図は対象外です（帯に内接固定。将来拡張）。
- 値は `true` / `false` のみ。それ以外はエラーで停止します。

## プロジェクト構成

| ファイル | 役割 |
|---|---|
| `pyproject.toml` | パッケージ定義（依存・`md2pptx` コンソールスクリプト） |
| `md2pptx/cli.py` | CLI エントリポイント（`md2pptx = md2pptx.cli:main`） |

## テーマについて

- **フルテーマ型の thmx**（スライドマスター＋レイアウト同梱）を前提とします。色・フォントのみの
  簡易テーマはレイアウトを持たないため利用できません（変換時にエラー）。
- thmx は内部で base pptx に変換されます。一度作った base pptx を手直しして固定運用したい場合は、
  その pptx を `--theme` に渡せます。

## 制限・既知の差

- `@table-widths` はスライド単位の指定で、多カラムでは全カラムの表に同じ比率が適用されます
  （列数が比率の要素数と一致しない表は等幅のまま）。カラム分割比は既定ではレイアウト依存
  （通常 50/50）ですが、`@widths` でスライド単位に変更できます。
- 表・図のフォントサイズや配置は概算見積もりに基づくため、実 PowerPoint の組版と完全一致は
  しません（安全側に縮小）。
- フロー図は横／縦一列のフローまで（分岐・格子は将来拡張）。
